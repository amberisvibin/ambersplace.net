@purpleraccoon5/29/2024, 4:06 AM

Windows tends to be...very controlling with boot loaders if you're booting in UEFI mode (if left to automatically partition a drive on a clean install, if it sees an EFI partition on another drive, even if something else is using it, it likes to use that which can indeed bork things up).

Generally I recommend that if you're starting fresh, wipe the OS drives (backing up anything on them you need first, of course), and then installing Windows first, and your linux distro on the other drive second. Most distros will have guided partition setups, and some may prompt you to use the existing EFI partition Windows has setup, but if you manually create an EFI parition (usually formatted FAT32 (which some installers will call just FAT)) and set its mount point /boot/efi and set that to Bootable (if the option is there), it should just use that. This way, the two OSes are completely separate, and if you blow away one drive the other OS will continue to work fine.

This also has the benefit of allowing you to use your motherboard boot menu to select which OS to boot from, and specify the default in the UEFI setup. If your distro uses GRUB or GRUB2, you can optionally (if it isn't enabled by default) set an option in grub's config called os-prober that will scan for other operating systems including Windows and automatically add a boot option for it, so you can just select either in the grub boot menu (timeouts can also be customized, and even the theme if you're interested).

If you have an existing linux install and want to install Windows after, just disconnect the linux drive physically, then install Windows on the empty drive, then once its totally done, shut down and reconnect the linux drive.

@purpleraccoon5/29/2024, 3:01 PM

Unfortunately, "all" in terms of hardware or software won't happen. There are still games that I play, and hardware that I use, where I need Windows for it (so, I dual boot).

Software wise, most games in Steam will work if you turn on the compatibility option to force even unofficially supported games to use Proton. For non-Steam games, I've had a lot of luck using Lutris, however there are always some that use kernel-mode anti-cheats that just refuse to work on linux no matter which compatibility solution you go with. Those will always need Windows.

With hardware, if there aren't kernel drivers, unfortunately the only way would be to either find someone willing to write them, or again either dual-boot Windows, or setup a Windows virtual machine with hardware passthru.

The unfortunate reality is that while I'd love to say "Everyone can move to linux!", that's only accurate if people are basically willing to re-buy a bunch of stuff (which sucks) or learn how to code and write their own software/drivers (which also sucks); both of these things are unrealistic and frankly not reasonable asks.

Especially when it comes to work stuff, you're probably best off sticking with Windows, and if you're curious about linux but don't want to risk work stuff or your existing setup, setting up a linux VM would probably be the safest way while still giving you the chance to get a feel for things.

@purpleraccoon6/2/2024, 6:37 AM

So, the nice thing about printers, sound, and other peripherals is that things are a hell of a lot better than they were even 10 years ago. Sound largely just works unless you're using bleeding edge hardware on certain laptops (and even that's been pretty much sorted, as mine started working with kernel 6.8), and printers connected via USB also tend to pretty much just work (as long as you have CUPS installed, and sometimes an extra package like hplip for HP printers, depending on the manufacturer and the distro you're using).

Games are also largely straightforward...ish. With Steam, you can enable Proton for even unsupported titles so most Windows games are playable, and non-steam games can often work with Lutris (there are usually templates available on their site to autoconfigure them). Games that have DRM that ties into the Windows kernel are a no-go, tho.

So...in terms of an actual distro, that's the hardest easy question ever.

My distro of choice these days is OpenSUSE Tumbleweed. I prefer it because it's a rolling release, and I don't have to do big upgraded to major new versions every six months, but it's also more prone to breaking on certain updates. Not something I generally recommend unless you're really comfortable with an RPM package system and getting your hands dirty unfucking things that lead to issues booting.

Debian is an easy recommendation, as long as you aren't running bleeding edge hardware, as the kernel and packages are Stable, but Old (even if you choose Debian Unstable, which isn't really unstable, just less aggressively tested than the Stable release). As there are so many distros based on it, it's usually pretty safe to assume that most software will be packaged for it, but again, bleeding edge stuff may pose a problem.

I really can't in good conscience recommend Ubuntu anymore, as Canonical is turning into the Microsoft of the linux world (I'd need an essay on that one), but I do recommend Mint, which is based on Ubuntu, and and especially if you have an Nvidia GPU - it's one of the best distros to get quickly up-and-running with that hardware).

Fedora...doesn't really get my recommendation anymore. It's widely supported, sure, but every time I use it I have more trouble than it's worth and switch back to OpenSUSE (which again I should mention is RPM-based like Fedora, and a lot of things that say they only support Fedora but provide an RPM will usually just work on OpenSUSE as well).

Most distros offer Spins/variations with different desktop environments, and if you've only ever really interacted with linux via SSH, I'd generally suggest starting with KDE, as it'll be most familiar and polished. All of the distros I've mentioned allow you to have multiple desktop environments installed at the same time, so even if you start there and want to experiment with others, it's usually pretty easy to install another, reboot, and at the login screen select another one.

So in summary, Mint is an excellent place to start, especially if you have Nvidia hardware. I do recommend OpenSUSE (either Tumbleweed for very fast updates, or Slowroll for a more stable setup) if you're already comfortable enough to get your hands dirty.

@purpleraccoon6/3/2024, 2:27 PM

If you have HP printers, installing the hplip package will let you add them (I have one on the network and just use the IPP address (ipp://(ip)). For non-HP, I usually just use "zypper se epson" to search for the drive package.

Autodiscovery isn't something that I tend to want/use, but OpenSUSE should come with cups-filters, which provides cups-browsed, which is ultimately the service that Mint/Ubuntu enable out-of-box to do this. You can get it up on OpenSUSE with:

sudo systemctl enable cups-browsed
sudo systemctl start cups-browsed

and it should start listening for advertisements from printers (yay for nearly all even semi-modern network printers having cups servers built-in) and add appropriately.
